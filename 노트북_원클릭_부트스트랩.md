# 🚀 회사 노트북 — 원클릭 부트스트랩 프롬프트

회사 노트북의 Claude Code (또는 Claude Desktop) 에서 **아래 프롬프트 한 덩어리를 복사해서 붙여넣고 Enter** 만 누르면 됩니다.
Claude 가 알아서 환경 점검 → clone → install → .env 셋업 → 세션 안내까지 진행합니다.

---

## 📋 복사해서 붙여넣을 프롬프트 (👇 아래 줄부터 끝까지 모두 선택)

```
컨버전스 2본부 실적관리 프로젝트를 이 PC 에 셋업해줘. 너는 전문가이고 다음을 자동으로 진행해.

[프로젝트 정보]
- GitHub: https://github.com/parkh37t/con-mgt
- 기술스택: Vanilla JS SPA + Express + Supabase Postgres
- 배포: master 푸시 → Vercel + Render 자동
- 브랜드: Wylie purple #3A2A6B

[자동 실행 순서]

1단계 · 환경 점검
PowerShell 또는 bash 에서 다음 확인:
- git --version  (없으면 https://git-scm.com/download/win 안내)
- node --version (없으면 https://nodejs.org LTS 안내, v18+)
- npm --version
- 사용자 홈 디렉토리 위치 확인 ($env:USERPROFILE 또는 $HOME)

2단계 · 작업 폴더 결정
나에게 한 번 물어봐: "기본 경로 C:\Users\<나>\Documents\con-mgt 에 클론할까요? (Enter=예, 다른 경로 입력 가능)"
이미 폴더가 존재하면 → "기존 폴더 발견. git pull 로 최신화할게요" 하고 4단계로 점프.

3단계 · 저장소 clone + 의존성 설치
cd <부모폴더>
git clone https://github.com/parkh37t/con-mgt.git
cd con-mgt
npm install
(약 1-2분 소요. 진행상황 알려줘)

4단계 · .env 파일 확인
con-mgt/.env 가 존재하면 → "OK, 이미 있음" 하고 5단계
없으면 → 다음 메시지 표시 후 사용자 응답 기다리기:

   ⚠️ .env 파일 필요 (DB 연결용)
   다음 두 방법 중 선택해주세요:
   A) 기존 PC OneDrive 에서 복사 — OneDrive\_secrets\con-mgt.env 위치에 두셨다면
      → 자동으로 복사할게요 ("A" 입력)
   B) 직접 입력 — 4가지 값 알려주시면 .env 만들겠습니다 ("B" 입력)
      필요값: DATABASE_URL, JWT_SECRET, ADMIN_USERNAME, ADMIN_PASSWORD

A 선택 시:
   copy "%USERPROFILE%\OneDrive\_secrets\con-mgt.env" .env
   파일 없으면 → "OneDrive 동기화 대기 중일 수 있습니다. OneDrive 아이콘 우클릭으로 동기화 확인 후 다시 시도하거나 B 방법 선택해주세요"

B 선택 시:
   각 값을 하나씩 물어보고 .env 작성

5단계 · 셋업 완료 안내
✅ 셋업 완료! 다음을 알려줘:
- 클론된 위치 (전체 경로)
- node_modules 설치 완료
- .env 설정 상태
- 다음 행동: 좌측 사이드바 「Pinned」 의 「컨버전스 2본부 실적관리」 세션 클릭 → 데스크탑 PC 작업 컨텍스트 그대로 이어받기
- 안 보이면: 같은 Anthropic 계정인지 확인, F5 새로고침, 30초 대기

6단계 · (선택) 로컬 미리보기 실행 여부 묻기
"로컬에서 npm run dev 로 미리보기 실행해볼까요? (Enter=skip, 'y'=실행)"

[작업 시 참고할 핵심 컨텍스트]
- 메인 코드: public/index.html (~660KB Vanilla JS SPA)
- 인증: routes/auth.js (JWT + bcrypt + 회원관리)
- DB: db.js (Postgres + 마이그레이션 v16)
- 데이터 모델: data.projects[], data.execPnL[pid], data.manpower.people[], data.projects3
- 핵심 함수: _buildExecMatrix(p) - 단일 진실 P&L 매트릭스
- 매출 수정: execEditRevenueByMonth(monthKey, value) → revenueOverride
- 정직원 picker: openEmployeePicker(rowKey, role) → pickEmployeeForExec(idx)
- 현행화: execRefreshFromSchedule() (드래프트 → 본 데이터)
- 워크플로: master 직접 푸시, .env 절대 커밋 금지, 작업 시작 시 git pull 먼저

[푸시 정책]
앞으로 모든 git push 는 사용자에게 물어보지 않고 자동 진행 (master 직접 푸시 OK)

[관련 문서] 같은 Cowork 세션 폴더에 있음:
- con-mgt_노트북_이어받기_가이드.docx
- con-mgt_전체프롬프트_아카이브.docx
- con-mgt_다른PC_작업가이드.docx

자, 1단계부터 시작해줘.
```

## 💡 사용 방법

### Step 1: 회사 노트북에서 Claude Code 또는 Claude Desktop 실행
- 데스크탑 PC 와 **같은 Anthropic 계정** 으로 로그인되어 있어야 함

### Step 2: 좌측 사이드바 「Pinned」 → 「컨버전스 2본부 실적관리」 클릭
- 세션이 자동 동기화되어 있을 것
- 안 보이면 30초 기다리거나 새 세션 만들고 진행

### Step 3: 위 프롬프트 박스 안의 내용 전체를 복사

### Step 4: Claude Code 입력창에 붙여넣고 Enter

### Step 5: Claude 가 알려주는 단계별 안내 따라가기
- 환경 점검 → 폴더 결정 → clone → install → .env 셋업 → 완료 안내
- 약 5-10분 소요 (네트워크 속도에 따라)

## 🆘 문제 해결

### Claude 가 응답을 안 한다
- Claude Code 가 코드 실행 권한이 있는지 확인 (PowerShell/Bash 도구 사용 가능 여부)
- 권한 요청 팝업이 떴다면 "허용" 클릭

### 권한 거부로 git push 가 막힌다
- 현재 정책: master 직접 푸시 자동 허용
- 만약 거부되면: "앞으로 모든 푸시는 자동 진행해줘" 라고 한 번 더 명시

### .env 가 OneDrive 에 없다
- 먼저 데스크탑 PC 에서:
  ```
  copy "C:\Users\jaeha\Documents\Cowork\con-mgt\.env" "C:\Users\jaeha\OneDrive\_secrets\con-mgt.env"
  ```
- OneDrive 동기화 후 다시 시도

### 세션이 사이드바에 안 보인다
- Cowork 동기화 30초~1분 소요 가능
- 데스크탑 PC 에서 아무 메시지 한 줄 보내고 다시 회사 노트북에서 새로고침
- 그래도 안 보이면: 새 세션 만들고 위 부트스트랩 프롬프트 사용 (이전 컨텍스트 없이 새로 시작)
